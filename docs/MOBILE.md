# Refuel 모바일 챕터 블루프린트 (1.0 로드맵)

> 목표: "앱스토어에서 받는 진짜 앱". 현재 자산 = 폰 PWA(암호화 동기화·설치형) + ntfy 푸시.
> 원칙: 이미 검증된 PWA를 최대한 재사용, 서버비 0원 구조 유지.

## 경로 비교

| 경로 | 스토어 | 비용 | 개발량 | 푸시 | 판단 |
|---|---|---|---|---|---|
| **M1. PWA 강화** (현재) | 스토어 불필요, 홈화면 설치 | 0원 | ✅ 완료 | ntfy 앱 병행 | 지금 당장 사용 가능 |
| **M2. TWA (Bubblewrap)** | **Google Play** | $25 1회 | 작음 — 기존 PWA를 그대로 감쌈 | M3에서 FCM 통합 | **다음 단계 추천** |
| M3. Capacitor 하이브리드 | Play + App Store | $25 + 연$99 | 중간 — 웹뷰 + 네이티브 푸시(FCM/APNs) 플러그인 | 자체 푸시(ntfy 앱 불필요) | 반응 확인 후 |
| M4. 풀 네이티브(RN/Flutter) | 둘 다 | 동일 | 큼 — UI 재작성 | 자체 | 상용 확장 때만 |

## 추천 시퀀스
1. **M1 (완료)**: 매니페스트/아이콘/오프라인/iOS 메타 — 홈화면 설치가 네이티브급.
2. **M2**: Play 개발자 계정($25) → Bubblewrap으로 TWA 패키징 → 비공개 테스트 트랙 → 출시.
   - 필요한 것: 개발자 계정(사용자), assetlinks.json(이 repo docs에 추가), 서명키.
   - PWA가 곧 앱이라 **기능 추가 = git push만으로 앱도 업데이트**.
3. **M3 (iOS 갈 때)**: Capacitor로 감싸고 푸시만 네이티브(FCM/APNs)로 교체.
   - iOS는 Apple 개발자 계정(연 $99) + 빌드용 Mac(또는 클라우드 CI) 필요.
   - 이 시점에 ntfy 의존 제거 가능(수집기 → FCM 직접). 서버는 여전히 0~수천원(FCM 무료).

## iOS 참고 (PWA 상태로도)
- Safari 공유 → "홈 화면에 추가" = 전체화면 앱으로 동작 (apple-touch-icon 적용됨)
- iOS 16.4+ 는 홈화면 PWA 웹푸시도 지원 — M3 전에 ntfy 없이 푸시하는 실험 여지 있음
